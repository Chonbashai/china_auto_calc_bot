# Telegram-бот: калькулятор авто из Китая

Пошаговая инструкция по установке на ваш сервер Beget.

Этот бот считает стоимость автомобиля из Китая «под ключ»: берёт курс юаня с сайта ВТБ, считает таможню через Calcus и показывает итог в Telegram.

---

## Что вам понадобится заранее

Перед началом подготовьте:

1. **Сервер Beget VPS** — у вас уже есть, Docker на нём установлен.
2. **Домен** — например `chinaauto.ваш-сайт.ru` (привязан к серверу).
3. **Nginx Proxy Manager** — у вас уже работает на сервере.
4. **PostgreSQL** — уже запущен на сервере (отдельный контейнер, например `everest-db`).
5. **Токен Telegram-бота** — получите у [@BotFather](https://t.me/BotFather) (инструкция ниже).
6. **Ваш Telegram ID** — для доступа к команде `/stats` (инструкция ниже).
7. **Доступ к серверу по SSH** — логин и пароль (или ключ) от Beget.

> Важно: мы **не трогаем** другие проекты на сервере (n8n, everest-crm и т.д.). Бот ставится в отдельную папку и в отдельный Docker-контейнер.

## Шаг 1. Создайте бота в Telegram

1. Откройте Telegram, найдите **@BotFather**.
2. Отправьте команду: `/newbot`
3. Придумайте имя бота (например: `Калькулятор авто из Китая`).
4. Придумайте username (должен заканчиваться на `bot`, например: `china_auto_calc_bot`).
5. BotFather пришлёт **токен** — длинная строка вида:
   ```
   7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
6. **Скопируйте и сохраните** этот токен — он понадобится в Шаге 5.

---

## Шаг 2. Узнайте свой Telegram ID (для админки)

1. Откройте в Telegram бота **@userinfobot** (или **@getmyid_bot**).
2. Нажмите **Start**.
3. Бот пришлёт ваш **Id** — число вида `123456789`.
4. **Скопируйте и сохраните** — это ваш `ADMIN_IDS`.

Если админов несколько — запишите все ID через запятую: `123456789,987654321`

## Шаг 3. Создайте базу данных

Бот хранит историю расчётов в PostgreSQL. Нужна **отдельная** база — существующие базы не трогаем.

### Вариант А: через Portainer (проще, если не любите командную строку)

1. Откройте **Portainer** в браузере (у вас он уже установлен).
2. Найдите контейнер с PostgreSQL (например `everest-db`).
3. Зайдите в контейнер → **Console** / **Exec**.
4. Введите команду:
   ```
   psql -U postgres
   ```
   (если у вас другой пользователь — подставьте его вместо `postgres`)
5. В открывшейся консоли PostgreSQL введите:
   ```sql
   CREATE DATABASE china_auto_bot;
   ```
6. Если всё прошло без ошибки — напишите `\q` и нажмите Enter (выход).

### Вариант Б: через SSH (командная строка)

1. Подключитесь к серверу (см. Шаг 4, пункт 1).
2. Выполните:
   ```bash
   docker exec -it everest-db psql -U postgres -c "CREATE DATABASE china_auto_bot;"
   ```
   > Замените `everest-db` на имя вашего PostgreSQL-контейнера, если оно другое.  
   > Посмотреть список контейнеров: `docker ps`

Таблицы бот создаст **сам** при первом запуске. Вручную создавать таблицы не нужно.

## Шаг 4. Загрузите проект на сервер

### 4.1. Подключитесь к серверу по SSH

**Windows (PowerShell или PuTTY):**
```bash
ssh root@ВАШ_IP_СЕРВЕРА
```
Подставьте IP вашего VPS из панели Beget.

**Mac / Linux (Терминал):** та же команда.

Введите пароль, когда попросит.

### 4.2. Скачайте проект с GitHub

Если проект уже на GitHub — выполните на сервере:

```bash
cd /opt
git clone https://github.com/ВАШ_ЛОГИН/china-auto-bot.git china-auto-bot
cd china-auto-bot
```

> Если репозитория на GitHub ещё нет — сначала загрузите папку проекта на GitHub с компьютера, потом выполните команду выше.

### 4.3. Альтернатива: загрузить папку вручную

Если Git пока не используете:

1. Заархивируйте папку `bot raschet` в ZIP на компьютере.
2. Загрузите ZIP на сервер через FileZilla / WinSCP в папку `/opt/china-auto-bot`.
3. На сервере распакуйте:
   ```bash
   cd /opt
   unzip china-auto-bot.zip -d china-auto-bot
   cd china-auto-bot
   ```

## Шаг 5. Настройте файл .env

Файл `.env` — это настройки бота. Без него бот не запустится.

### 5.1. Создайте файл из шаблона

На сервере, в папке проекта:

```bash
cp .env.example .env
nano .env
```

Откроется редактор. Заполните **каждую** строку:

```env
BOT_TOKEN=7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
DATABASE_URL=postgresql://root:123Qwe321@n8n-stack-postgres-1:5432/china_auto_bot
WEBHOOK_URL=https://chinaauto.ваш-сайт.ru/telegram/webhook
ADMIN_IDS=123456789
PORT=3000
NODE_ENV=production
```

**Что писать в каждой строке:**

| Строка | Что вписать | Пример |
|--------|-------------|--------|
| `BOT_TOKEN` | Токен от BotFather (Шаг 1) | `7123456789:AAH...` |
| `DATABASE_URL` | Адрес базы данных | см. ниже |
| `WEBHOOK_URL` | Адрес бота в интернете | `https://ваш-домен.ru/telegram/webhook` |
| `ADMIN_IDS` | Ваш Telegram ID (Шаг 2) | `123456789` |
| `PORT` | Не меняйте | `3000` |
| `NODE_ENV` | Не меняйте | `production` |

### 5.2. Как составить DATABASE_URL

Формат:
```
postgresql://ЛОГИН:ПАРОЛЬ@ХОСТ:5432/china_auto_bot
```

**Рекомендуется (бот и PostgreSQL в одной Docker-сети):**
```
postgresql://root:123Qwe321@n8n-stack-postgres-1:5432/china_auto_bot
```

- `root` — логин PostgreSQL (уточните у того, кто настраивал сервер)
- `123Qwe321` — пароль от PostgreSQL
- `n8n-stack-postgres-1` — **имя postgres-контейнера** внутри Docker-сети (не IP)
- `china_auto_bot` — имя базы из Шага 3

Контейнер бота подключается к БД в общей Docker-сети (`n8n-stack_default`). Это устраняет ошибку `ETIMEDOUT` при старте.

> Если имя postgres-контейнера или сеть другие — проверьте на сервере:
> ```bash
> docker network ls
> docker network inspect n8n-stack_default
> docker ps --format "{{.Names}}" | grep postgres
> ```
> Подставьте актуальное имя контейнера в `DATABASE_URL` и при необходимости обновите сеть в `docker-compose.yml`.

> Запасной вариант (если имя контейнера неизвестно): `172.17.0.1` или `everest-db` — но в production лучше использовать имя контейнера в общей сети.

### 5.3. Сохраните файл

В редакторе `nano`:
- **Ctrl + O** → Enter (сохранить)
- **Ctrl + X** (выйти)

## Шаг 6. Настройте домен в Nginx Proxy Manager

Пока домен не настроен — Telegram не сможет отправлять сообщения боту.

1. Откройте **Nginx Proxy Manager** в браузере (обычно `http://ВАШ_IP:81`).
2. Войдите под своим логином.
3. Перейдите: **Hosts** → **Proxy Hosts** → кнопка **Add Proxy Host**.

**Вкладка Details:**
- **Domain Names:** `chinaauto.ваш-сайт.ru` (ваш реальный домен)
- **Scheme:** `http`
- **Forward Hostname / IP:** `172.17.0.1` (или IP сервера)
- **Forward Port:** `3000`
- Включите: **Block Common Exploits**
- Включите: **Websockets Support**

**Вкладка SSL:**
- **SSL Certificate:** Request a new SSL Certificate
- Включите: **Force SSL**
- Включите: **HTTP/2 Support**
- Email: ваш email (для Let's Encrypt)
- Согласитесь с условиями → **Save**

4. Подождите 1–2 минуты, пока выпустится SSL-сертификат.

5. **Проверьте**, что в `.env` указан тот же домен:
   ```
   WEBHOOK_URL=https://chinaauto.ваш-сайт.ru/telegram/webhook
   ```
   Домен в `WEBHOOK_URL` и в NPM **должны совпадать**.

## Шаг 7. Запустите бота

На сервере, в папке проекта:

```bash
cd /opt/china-auto-bot
docker compose up -d --build
```

Первая сборка занимает **5–15 минут** — это нормально.

### Проверьте, что контейнер запустился

```bash
docker ps
```

В списке должна быть строка `china-auto-bot` со статусом **Up** (через минуту — **healthy**).

### Посмотрите логи (если что-то не работает)

```bash
docker compose logs -f china-auto-bot
```

Должны быть строки:
- `Application started on port 3000`
- `Webhook registered` или `Webhook already registered`

Выйти из логов: **Ctrl + C**

## Шаг 8. Проверьте, что всё работает

Пройдите по списку — ставьте галочки:

- [ ] **1.** Контейнер запущен: `docker ps` показывает `china-auto-bot`
- [ ] **2.** Сайт отвечает: откройте в браузере `https://ваш-домен.ru/health` — должно показать `{"status":"ok",...}`
- [ ] **3.** Бот отвечает в Telegram: отправьте `/start` — появится приветствие и кнопка «Рассчитать стоимость»
- [ ] **4.** Расчёт работает: нажмите кнопку, пройдите 5 вопросов, получите итоговую сумму
- [ ] **5.** История работает: отправьте `/history` — видны прошлые расчёты
- [ ] **6.** Статистика: отправьте `/stats` — видна статистика (только у вас, как админа)
- [ ] **7.** Чужой пользователь **не** видит `/stats` (попросите кого-нибудь проверить)

### Как пользоваться ботом

1. `/start` — начало работы
2. Кнопка **«Рассчитать стоимость»**
3. Бот спросит по очереди:
   - марку и модель (можно пропустить)
   - год выпуска
   - объём двигателя (см³)
   - мощность (кВт)
   - себестоимость в юанях
4. Бот сам посчитает и пришлёт результат
5. `/history` — последние 10 ваших расчётов

## Если что-то пошло не так

### Бот не отвечает в Telegram

1. Проверьте логи:
   ```bash
   docker compose logs china-auto-bot
   ```
2. Убедитесь, что `WEBHOOK_URL` в `.env` **точно** совпадает с доменом в NPM.
3. Проверьте SSL: домен должен открываться по `https://` без ошибок.
4. Перезапустите:
   ```bash
   docker compose restart china-auto-bot
   ```

### Ошибка подключения к базе данных

1. Проверьте, что база `china_auto_bot` создана (Шаг 3).
2. Проверьте логин и пароль в `DATABASE_URL`.
3. Убедитесь, что бот в той же Docker-сети, что и PostgreSQL:
   ```bash
   docker network inspect n8n-stack_default
   ```
   В списке контейнеров должны быть и `china-auto-bot`, и postgres-контейнер.
4. В `DATABASE_URL` используйте **имя postgres-контейнера** (например `n8n-stack-postgres-1`), а не внешний IP.
5. Если сеть или имя контейнера другие — обновите `docker-compose.yml` и `.env` по результатам `docker network ls` и `docker ps`.

### Не получается курс ВТБ или таможня Calcus

Это нормально — сайты иногда недоступны. Бот **сам попросит** ввести курс или таможню вручную. Просто введите число, когда бот спросит.

### Перезапустить бота

## Что делает бот (для справки)

Бот автоматически:
- берёт курс юаня с сайта ВТБ;
- считает таможню через calcus.ru;
- прибавляет фиксированные расходы (комиссия, брокер, лаборатория, прописка);
- сохраняет историю расчётов;
- показывает статистику администратору.

Формула итога:
```
Инвойс = себестоимость (¥) × курс × 1,025
Итого = инвойс + таможня + 65 000 + 50 000 + 50 000 + 10 000
```
